Microsoft To Do Authorization

Connect Reminder Gen

Microsoft requires every app that asks for To Do access to be registered first. This page keeps that one-time setup out of the task popup. Do not buy Entra ID P1/P2 for this prototype.

1. Register The App

Create a Microsoft Entra app registration and paste its client ID below.

Open Entra
  1. If Microsoft shows Entra ID P1/P2 pricing, back out. Those paid plans are not required for this extension.
  2. Go to Identity / Applications / App registrations.
  3. Click New registration.
  4. For personal To Do accounts, choose Accounts in any organizational directory and personal Microsoft accounts.
  5. After registering, copy Application (client) ID from the app Overview page.
  6. Add the Redirect URI below under Authentication as a Single-page application redirect URI. Do not add it as Web.
  7. Add delegated Microsoft Graph permissions: User.Read and Tasks.ReadWrite.

2. Local Settings

Settings are stored locally in this Chrome profile.

Use consumers for a personal Outlook/Hotmail Microsoft To Do account. Use organizations or your Directory tenant ID for a work or school account. Do not use common with a consumer-only app registration; Microsoft rejects that combination. A tenant ID is on the Microsoft Entra Overview page as Directory (tenant) ID. The paid Entra plans are for enterprise identity features, not for this local OAuth client.
